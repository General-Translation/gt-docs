# General Translation

Welcome to the docs for [General Translation](https://www.generaltranslation.com).

Our products:

- `gt-react`, an internationalization ("i18n") library for [React](https://react.dev) and [Next.js](https://nextjs.org) applications.

Unlike other i18n stacks, `gt-react`:

- **Translates automatically and on-demand** â€” once connected to our cloud service, your app will instantly support dozens of languages, no extra work needed.
- **Translates entire React components, not just strings.** So there's no need to rewrite your site to accomodate new languages.
- **Works across both client and server components**, with first-class support for Next.js App Router and React Server Components.

## Patterns

`gt-react` enables clean, efficient code, however you structure your content.
- If you need a quick, easy way to translate server components, it may be better to use the **Component Children Pattern**.
- If you're already storing your translatable content in a `.json` or `.js` file which you import throughout your project, or you have a lot of client components, it may be better to use the **Dictionary Pattern**. 

### Component Children Pattern

The component children pattern involves passing your translatable content as the children of a server-side `<T>`, `<Numeric>`, or `<Value>` component.
For example:

```js copy
import { createGT } from 'gt-react/next';

export const { T } = createGT({
    defaultLocale: 'en-US' // the language of your app
});
```

```js copy
import { T } from '@i18n'

export default function Page() {
    return (
        <main>
            <T id='welcome'>
                <h1>Welcome to my site!</h1>
                <p>This text will be <b>translated</b> into dozens of languages automatically.</p>
            </T>
        </main>
    )
}
```

Components which use client-side JavaScript, like React hooks, are client components. 
You can still pass a translation from server to client with a `<T>` component, like:

```js
import { T } from '@i18n'
import ClientSideButton from './ClientSideButton.js' // a client component

export default function Page() {
    return (
        <T id='landing.page'>
            <main>
                <h1>Welcome to my site!</h1>
                <ClientSideButton>Get Started!</ClientSideButton>
            </main>
        </T>
    )
}
```

However, sometimes it is difficult to pass children directly from server to client, for example if the client component is displayed conditionally, or is nested within another client component.
In this case, we recommend using a [`<GTProvider>`](gt-react/client/gtprovider).

### Dictionary Pattern

The dictionary pattern involves storing your content in a central file that looks like something like:

```jsx
// dictionary.js

const dictionary = {
    "landing": {
        "hero": <h1>Welcome to our site!</h1>,
        "get_started_button": (
            <a href='/signin'><button>Get Started</button></a>
        )
    },
    "client_component" {
        "greeting": <>Hello, world. This content is rendered in a client component.</>
    }
}

export default dictionary;
```

Then importing this file and passing it as a parameter to `createGT()`.

```javascript
// i18n.js
import dictionary from './dictionary.js'
import { createGT } from 'gt-react/next'

export const { t, GTProvider } = createGT({
    defaultLocale: 'en-US'
    dictionary: dictionary
})
```

In your components you can then import the `t()` function. Every piece of content rendered by the `t()` function is automatically translated.

```javascript
import { t } from '@i18n'

export default function Page() {
    return (
        <main>
            {t('landing.hero')}
            {t('landing.get_started_button')}
        </main>
    )
}
```

To pass these translations to client components, wrap them in a `<GTProvider>`.
The dictionary from the `i18n.js` config file is passed automatically:

```js copy
import { GTProvider } from '@i18n'

import ClientComponent from './ClientComponent.js'

export default function Page() {
    return (
        <GTProvider>
            <ClientComponent />
        </GTProvider>
    )
}
```

```js copy
'use client'

import { useGT } from 'gt-react/client'

export default function ClientComponent() {

    const t = useGT('client_component');

    return (
        <main>  
            {t('greeting')}
        </main>
    )
}
```
