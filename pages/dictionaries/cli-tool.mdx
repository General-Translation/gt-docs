# `gt-react-cli`

The `gt-react-cli` tool is used to push a JSX translation dictionary to General Translation's cloud cache.

The dictionary will be translated into your project's previous languages, plus any you specify while running the script, and be available and ready to use within a few minutes.

## Installation

To install the CLI as a development dependency, run:

```bash
npm i gt-react-cli --save-dev
```

## Usage

Run the following command from the **root of your repository**:

```bash
npx gt-react-push <path_to_dictionary> [--apiKey=<apiKey>] [--projectID=<projectID>] [--dictionaryName=<dictionaryName>] [--languages=<lang1,lang2,...>]
```

### Example

```bash
npx gt-react-push ./src/dictionary.js --apiKey=gtx-12..89 --projectID=ab..12 --dictionaryName=my-site --languages=fr,es
```

This command:
- Pushes the dictionary located at `./src/dictionary.js`.
- Uses the API key `gtx-12..89` and project ID `ab..12`.
- Updates the dictionary with the name `my-site`.
- Translates to French (`fr`) and Spanish (`es`).

### Required Arguments

- `--apiKey=<apiKey>`: The GeneralTranslation API key. 
- `--projectID=<projectID>`: The project ID for your translations.

Alternatively, place your `GT_API_KEY` and `GT_PROJECT_ID` in a `.env` file in the **root of your repository**.

```.env
GT_API_KEY="gtx-12..89"
GT_PROJECT_ID="ab..12"
```

### Optional Arguments

- `--dictionaryName=<dictionaryName>`: Specify the dictionary name to be updated.
- `--languages=<lang1,lang2,...>`: Comma-separated list of target languages (e.g., `fr`, `es`).

### Full Example

```bash
npx gt-react-push ./src/dictionary.js --dictionaryName=my-site --languages=fr,es
```

This will push the dictionary and update the translations for the specified languages.
