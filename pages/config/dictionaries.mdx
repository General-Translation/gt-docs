# Writing a dictionary

Exporting translatable content from a central dictionary file helps maintain consistency across your app — when a piece of text needs to be updated, you only need to change it in the central dictionary file.

Dictionaries in `gt-react` are written in your default language and accessed through the [`t()`](/essentials) function.

```js copy
// dictionary.js

const myDictionary = {
    "my_id": "Hello, world!"
}

export default myDictionary;
```

```js copy
// i18n.js
// initialize gt-react to translate myDictionary to any language

import myDictionary from './dictionary'
import { initGT } from 'gt-react/next'

// use the t() exported from this file in your components
export const {
	t, GTProvider
} = initGT({
	defaultLocale: "en",
	approvedLocales: [
		"en", "fr", "de", "es", "it"
	],
	dictionary: myDictionary
});
```

```js copy
// page.js

import { t } from '@i18n'

export default function Page() {
    return (
        <main>
            {
                t('my_id') /* will display the translation of "Hello, world" */
            }
        </main>
    )
}
```

## JSX

`gt-react` dictionaries can be written in arbitrary [JSX](https://react.dev/learn/writing-markup-with-jsx) (JavaScript XML). JSX is what you use to write your components in regular React, so you can just copy and paste your content into the dictionary file and it will work!

For example:

```jsx copy
// dictionary.js

const dictionary = {
    "string_entry": "This is a string!",
    "jsx_entry": <p>This is JSX with nested <b>bold</b> tags!</p>
    "jsx_entry_two": <h1 className="text-xl">This JSX even has a className!</h1>
}

export default dictionary;
```

## Imports

In `gt-react` dictionaries, you can import and use arbitrary components inside your dictionary. Only the text passed as children to the imported components will be translated by `gt-react`. Text generated internally by the components will **not** be translated.

```jsx
// dictionary.js

import CustomComponent from './CustomComponent';

const dictionary = {
    "string_entry": "This is a simple string!",
    
    // This text will be translated
    "jsx_entry": <p>Hello, world!</p>, 
    
    // The text "Hello, world!" will be translated because it's passed as children
    "jsx_with_custom_component_children": (
        <CustomComponent>Hello, world!</CustomComponent> 
    ),
    
    // The text "This text will be translated." will be translated, but not the text generated inside CustomComponent
    "jsx_with_import": (
        <div>
            <p>This text will be translated.</p>
            <CustomComponent /> {/* The content inside this component will NOT be translated */}
        </div>
    ),
};

export default dictionary;
```

**Translated:**
  - Plain text or JSX elements in the dictionary directly, like `<p>Hello, world!</p>`.
  - Text passed as children to an imported component, like `<CustomComponent>Hello, world!</CustomComponent>`.
  
**Not Translated:**
  - Text generated inside an imported component, such as the output of `CustomComponent` if it produces its own text (like a string `Hello, world!` created within the component itself).

## Context

You can provide additional context to help our AI system understand how certain terms or phrases should be translated. This is useful for words that may have multiple meanings. The context is added as an attribute in the second object in an array:

```jsx
// dictionary.js

import CustomComponent from './CustomComponent';

const dictionary = {
    "string_entry": "This is a simple string!",

    "jsx_entry": <p>Hello, world!</p>, 
    
    // Adding context to provide clarity for translation
    "plans": [
        "Plans", 
        { context: 'As in, "payment plans"' }
    ],
    
    // JSX entry with context
    "button_with_context": [
        <h1>Your favorite pics</p>,
        { context: 'Translate this informally' }
    ]
};

export default dictionary;
```

## Nested dictionaries

In `gt-react`, you can structure your dictionary into nested objects, allowing you to group related translations together. This approach is useful for organizing translations by pages, components, or sections. Nested translations are accessed using dot notation with the `t()` function.

```js copy
// dictionary.js

const dictionary = {
    "home_page": {
        "title": <h1>Welcome to the Home Page</h1>,
        "description": <p>This is the home page of our app.</p>
    }
};

export default dictionary;
```


```js copy
// page.js

import { t } from '@i18n'

export default function HomePage() {
    return (
        <main>
            {t('home_page.title')} {/* Will display <h1>Welcome to the Home Page</h1> */}
            {t('home_page.description')} {/* Will display <p>This is the home page of our app.</p> */}
        </main>
    )
}
```

## Variables

In `gt-react`, variables allow you to dynamically insert content into your translations that can change but shouldn’t be translated. For example, user names, dates, or numbers may vary between sessions or users, but the surrounding text remains constant. You can use the `initVariables()` function to create components like `<Var>` for inserting custom variables into your translations. Additionally, specialized variable types such as `<Num>`, `<DateTime>`, and `<Currency>` can handle localized numbers, dates, and currencies.

```js copy
// dictionary.js
import { initVariables } from 'gt-react/next';

const { Var } = initVariables();

const dictionary = {
    "greeting": <>Welcome to our site, <Var name="firstname"/>!</>
};

export default dictionary;
```

```js copy
// page.js

import { t } from '@i18n';

export default function Page() {
    const firstname = "Alice";

    return (
        <main>
            <p>{t('greeting', { firstname })}</p> {/* Displays "Welcome to our site, Alice!" */}
        </main>
    );
}
```

## Plurals

<Callout type="info">
    Pass `n` to handle plurals; the library will select the correct form based on the number provided.
</Callout>

`gt-react` also supports the automatic handling of plurals in translations. This will work out of the box in languages like Arabic or Welsh, which have more than just singular and plural forms — with no need for an English-speaking developer to provide extra info.

```js copy
// dictionary.js
import { initVariables } from 'gt-react/next';

const { Num } = initVariables();

const dictionary = {
    "message_count": [
        <>You have <Num/> new messages.</>,
        { singular: <>You have one new message.</> }
    ]
};

export default dictionary;
```

```js copy
// page.js

import { t } from '@i18n';

export default function Page() {
    const messageCount = 5;

    return (
        <main>
            <p>{t('message_count', { n: messageCount })}</p> {/* Displays "You have 5 new messages." */}
        </main>
    );
}
```
