## Overview

The `useLocale` hook retrieves the user's current locale from the [`<GTProvider>` context](/docs/next/api/components/gtprovider).
This is essential for determining the active language and locale settings of the application.
The returned locale is [formatted as a string]((/docs/reference/locale-strings)), e.g., `'en-US'`.

<Callout>
    `useLocale` is a client-side hook and *can only be used on client-side components*.
    For server-side locale management, see [getLocale](/docs/next/api/methods/server/getLocale).
</Callout>

### When to Use

- Use `useLocale` to get the active locale in client components.
- Use it to dynamically adjust content, styles, or behavior based on the user's language settings.

<Callout type="warn">
    The `useLocale` hook can only be used within a component wrapped by a [`<GTProvider>`](/docs/next/api/components/gtprovider).
    Attempting to use it outside of a provider will result in an error.
</Callout>

## Reference

### Returns

A string representing the user's current locale, e.g., `'en-US'`.

## Example

### Basic Usage

Retrieve the current locale and display it in your component.

```jsx title="src/components/CurrentLocale.jsx" copy
import { useLocale } from 'gt-next'; // [!code highlight]

export default function CurrentLocale() {
    const locale = useLocale(); // Get the user's locale

    return <p>Current locale: {locale}</p>; // Display the locale
}
```

## Adjusting Content Based on Locale

Use the `useLocale` hook to render locale-specific content dynamically.

```jsx title="src/components/LocalizedGreeting.jsx" copy
import { useLocale } from 'gt-next'; // [!code highlight]

export default function LocalizedGreeting() {
    const locale = useLocale(); // Get the user's locale // [!code highlight]

    return (
        <p>
            { locale === 'fr-FR'
                ? 'French is not supported!'
                : 'Hello, world!'}
        </p>
    );
}
```

## Notes
 * The `useLocale` hook relies on the [`<GTProvider>`](/docs/next/api/components/gtprovider) to access the context. Ensure your app is wrapped with a provider at the root level.
 * Use this hook for client-side locale based behavior.
 * Learn more about locale strings and how to read them here: [Locale Strings](/docs/reference/locale-strings).
