## Überblick

Der `useDefaultLocale` Hook ruft die Standard-Lokalisierung der Anwendung aus dem [`<GTProvider>` Kontext](/docs/react/api/components/gtprovider) ab.
Diese Lokalisierung stellt die Fallback-Sprache für Ihre App dar und wird typischerweise verwendet, wenn die bevorzugte Lokalisierung eines Benutzers nicht verfügbar ist.

<Callout>
    Stellen Sie sicher, dass Ihre App in einen [`<GTProvider>`](/docs/react/api/components/gtprovider) eingebettet ist.
</Callout>

Siehe [`<GTProvider>`](/docs/react/api/config/gtprovider) für die Konfiguration.
Wenn keine Standard-Lokalisierung angegeben ist, wird standardmäßig `'en-US'` verwendet.


## Referenz

### Rückgaben

Ein String, der die Standard-Lokalisierung der Anwendung darstellt, z.B. `'en-US'`.

---

## Beispiele

### Grundlegende Verwendung

Rufen Sie die Standard-Lokalisierung der Anwendung ab und zeigen Sie sie in Ihrer Komponente an.

```jsx title="DefaultLocale.jsx" copy
import { useDefaultLocale } from 'gt-react';

export default function DefaultLocale() {
    const defaultLocale = useDefaultLocale();// [!code highlight]

    return <p>Standard-Lokalisierung: {defaultLocale}</p>; // Display the default locale
}
```

## Notizen
 * Der `useDefaultLocale()` Hook basiert auf dem [`<GTProvider>`](/docs/react/api/components/gtprovider), um auf den Kontext zuzugreifen.
   Stellen Sie sicher, dass Ihre App auf der Root-Ebene mit einem Provider umschlossen ist.
 * Erfahren Sie mehr über Locale-Strings [hier](/docs/reference/locale-strings).
