## Überblick

Die Funktion `getDefaultLocale()` ruft die Standard-Lokalisierung der Anwendung ab.
Diese Lokalisierung wird beim Start des Servers festgelegt und dient als Fallback-Sprache für Übersetzungen.
Sie gibt einen [Lokalisierungs-String](/docs/reference/locale-strings) zurück, z.B. `'en-US'`.

<Callout>
    `getDefaultLocale()` ist eine serverseitige Methode und kann nur auf serverseitigen Komponenten verwendet werden.
</Callout>

Siehe [`withGTConfig()`](/docs/next/api/config/withGTConfig) für die Konfiguration.
Wenn in [`withGTConfig()`](/docs/next/api/config/withGTConfig) keine Standard-Lokalisierung angegeben ist, wird standardmäßig `'en-US'` verwendet.
Für die Client-Seite siehe [`useDefaultLocale()`](/docs/next/api/helpers/useDefaultLocale).

## Referenz

### Rückgaben

Ein String, der die Standard-Lokalisierung der Anwendung darstellt, z.B. `'en-US'`.


---



## Beispiele

### Grundlegende Verwendung

Rufen Sie die Standard-Lokalisierung der Anwendung ab und verwenden Sie sie in Ihrer serverseitigen Logik.

```jsx title="GetDefaultLocale.jsx" copy
import getDefaultLocale from 'gt-next/server';

export default function GetDefaultLocale() {
    const defaultLocale = getDefaultLocale(); // [!code highlight]
    return <p>Standard-Lokalisierung: {defaultLocale}</p>;
}
```

---


## Notizen
 * Die `getDefaultLocale()` gibt die Standard-Lokalisierung zurück, die in [`withGTConfig()`](/docs/next/api/config/withGTConfig) festgelegt ist.
 * `getDefaultLocale()` ist nur serverseitig.
 * Die zurückgegebene Lokalisierung entspricht dem Format der [Lokalisierungszeichenfolgen](/docs/reference/locale-strings).

## Nächste Schritte
 * Siehe [`useLocale()`](/docs/next/api/helpers/useLocale) und [`getLocale()`](/docs/next/api/helpers/getLocale), um die Spracheinstellung des Benutzers abzurufen.
