## Überblick

Dieses Tutorial zeigt Ihnen, wie Sie die Übersetzungsdateien Ihres Projekts in [`i18next`](https://i18next.com/) automatisch verwalten können.


Wir werden diese 4 Schritte befolgen:
<Steps>
  <Step>
    Fügen Sie Ihre Umgebungsvariablen hinzu
  </Step>
  <Step>
    Installieren Sie [`gtx-cli`](/docs/cli)
  </Step>
  <Step>
    Initialisieren Sie Ihr Projekt mit dem Befehl [`gtx-cli init`](/docs/cli/init)
  </Step>
  <Step>
    Führen Sie [`gtx-cli translate`](/docs/cli/translate#translate-json) aus
  </Step>
</Steps>

<Callout>
  **Tipp:**
  Vermeiden Sie den Aufwand mit Übersetzungsdateien durch die Verwendung der [`<T>` Komponente](/docs/usage/t).
</Callout>

---

## Schritt 1: Fügen Sie Ihre Umgebungsvariablen hinzu

Fügen Sie Ihren Produktions-API-Schlüssel und Ihre Projekt-ID zu Ihren Umgebungsvariablen hinzu.
Dies ist notwendig, um das `gtx-cli`-Tool zu verwenden.
Sie erhalten diese vom [General Translation Dashboard](https://generaltranslation.com/dashboard).

```bash title=".env"
GT_API_KEY=<your-api-key>
GT_PROJECT_ID=<your-project-id>
```

## Schritt 2: Installieren Sie `gtx-cli`

Installieren Sie das `gtx-cli`-Tool in Ihrem Projekt.

<Tabs items={['npm', 'yarn', 'bun', 'pnpm']}>
  <Tab value="npm">
  ```bash 
  npm i --save-dev gtx-cli
  ``` 
  </Tab>
  <Tab value="yarn">
  ```bash 
  yarn add --dev gtx-cli
  ```
  </Tab>

  <Tab value="bun">
  ```bash 
  bun add --dev gtx-cli
  ```
  </Tab>
  <Tab value="pnpm">
  ```bash 
  pnpm add --save-dev gtx-cli
  ```
  </Tab>
</Tabs>

## Schritt 3: Initialisieren Sie Ihr Projekt mit dem Befehl `gtx-cli init`

Führen Sie den Befehl `gtx-cli init` aus, um Ihr Projekt zu konfigurieren.

```bash
npx gtx-cli init
```

## Schritt 4: Fügen Sie den Befehl `gtx-cli translate` zu Ihrem Build-Prozess hinzu

Fügen Sie den Befehl `gtx-cli translate` zu Ihrem Build- oder CI-Prozess vor dem Build-Befehl hinzu, um automatisch Übersetzungen zu Ihrem Projekt hinzuzufügen.

```json title="package.json"
{
  "scripts": {
    "translate": "npx gtx-cli translate",
    "build": "npm run translate && <your build command>"
  }
}
```
Dies generiert Übersetzungen für alle Ihre Sprachen und speichert sie in Ihrem Projekt.
Wenn Sie diese Dateien in Ihr Repository übernehmen möchten, können Sie diesen Befehl stattdessen vor dem Commit ausführen.


Sie sind fertig! Jetzt wird Ihr Projekt automatisch alle Ihre Übersetzungs-JSON-Dateien aktualisieren, wenn sich Ihr Projekt ändert.

---

## Hinweise
 * Sie können Übersetzungen automatisch zu Ihrem Projekt hinzufügen mit dem Befehl [`gtx-cli translate`](/docs/cli/translate#translate-json).
 * Wenn Sie Ihre Übersetzungsdateien committen möchten, können Sie stattdessen den Befehl `gtx-cli translate` vor dem Commit ausführen.
 * Um den Ausgabepfad für Ihre Übersetzungen zu konfigurieren, siehe die [Konfigurationsdokumentation](/docs/cli/reference/config).

## Nächste Schritte
 * Sehen Sie sich die [Nutzungsanleitung](/docs/cli/reference/usage) für das CLI-Tool an.
 * Wenn Sie Live-Übersetzungen in der Entwicklung sehen möchten (wie Hot Reload), schauen Sie sich die [`<T>` Komponente](/docs/usage/t) an.
