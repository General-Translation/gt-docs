## 概要

`getDefaultLocale()` 関数はアプリケーションのデフォルトのロケールを取得します。
このロケールはサーバーが起動したときに決定され、翻訳のフォールバック言語として使用されます。
[ロケール文字列](/docs/reference/locale-strings)を返します。例: `'en-US'`。

<Callout>
    `getDefaultLocale()` はサーバーサイドのメソッドであり、サーバーサイドコンポーネントでのみ使用できます。
</Callout>

設定については [`withGTConfig()`](/docs/next/api/config/withGTConfig) を参照してください。
[`withGTConfig()`](/docs/next/api/config/withGTConfig) でデフォルトのロケールが指定されていない場合、デフォルトは `'en-US'` になります。
クライアントサイドについては、[`useDefaultLocale()`](/docs/next/api/helpers/useDefaultLocale) を参照してください。

## リファレンス

### 戻り値

アプリケーションのデフォルトのロケールを表す文字列、例: `'en-US'`。


---



## 例

### 基本的な使用法

アプリケーションのデフォルトのロケールを取得し、サーバーサイドのロジックで使用します。

```jsx title="GetDefaultLocale.jsx" copy
import getDefaultLocale from 'gt-next/server';

export default function GetDefaultLocale() {
    const defaultLocale = getDefaultLocale(); // [!code highlight]
    return <p>Default locale: {defaultLocale}</p>;
}
```

---


## メモ
 * `getDefaultLocale()` は [`withGTConfig()`](/docs/next/api/config/withGTConfig) で設定されたデフォルトのロケールを返します。
 * `getDefaultLocale()` はサーバーサイドのみです。
 * 返されるロケールは [ロケール文字列](/docs/reference/locale-strings) の形式に従います。

## 次のステップ
 * ユーザーのロケールを取得するには、[`useLocale()`](/docs/next/api/helpers/useLocale) および [`getLocale()`](/docs/next/api/helpers/getLocale) を参照してください。
 * クライアントコンポーネントでデフォルトのロケールを取得するには、[`useDefaultLocale()`](/docs/next/api/helpers/useDefaultLocale) を探索してください。
