## 概述

字典是在通用翻译中翻译字符串内容的一种方式。
所有内容都存储在一个中央文件中，可以使用[`useDict()` 钩子](/docs/react/api/dictionary/useDict)访问。

如果你使用过其他i18n库，你可能熟悉使用翻译文件（`en.json`、`zh.json`等）。
使用字典在语法上基本相同，只是你只需要维护一个基本语言的文件。
所有翻译将由LLMs生成。

<Callout>
  **提示**：
  如果你正在寻找一种更简单的方式来翻译字符串内容，我们建议使用[`useGT()` 钩子](/docs/react/api/strings/useGT)。
</Callout>

---

## 优点和缺点

使用字典有几个优点：

1. **集中存储**：字典将所有可翻译内容存储在单个文件中，使其更容易管理和更新。
2. **历史先例**：字典模式是行业中常见的设计模式，被许多其他库使用。

同时，它也有一些缺点：

1. **复杂性**：字典比内联翻译字符串内容更复杂设置和使用。
2. **可读性**：字典比内联翻译的可读性差，因为内容与使用它的上下文分离。

字典是内联翻译的替代设计模式，但这些模式都受到我们库的支持，并且不是互斥的。
你可以将字典与[`<T>` 组件](/docs/react/api/components/t)和[`useGT()` 钩子](/docs/react/api/strings/useGT)的内联翻译一起使用。

---

## 示例

在字典文件（`.js`或`.json`）中指定你的字符串内容。
```jsx title="src/dictionary.json" copy
{
  "greetings": {
    "hello": 'Hello, World!',
  },
} 
```

然后通过将键传递给`d()`函数在你的组件中引用它。

```jsx title="src/MyComponent" copy
import { GTProvider, useDict } from 'gt-react';
import dictionary from './dictionary.json';

export default function MyComponent() {

  const d = useDict(); // [!code highlight]

  return (
    <GTProvider dictionary={dictionary}>
      <div>
        {d('greetings.hello')} // [!code highlight]
      </div>
    </GTProvider>
  );
}
```

## 注意事项
 * 字典是在通用翻译中翻译内容的一种方式。
 * [`useDict()` 钩子](/docs/react/api/dictionary/useDict)用于使用字典模式翻译字符串。
