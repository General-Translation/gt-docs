## Aperçu

Le hook `useDefaultLocale` récupère la locale par défaut de l'application à partir du [contexte `<GTProvider>`](/docs/next/api/components/gtprovider).
Cette locale représente la langue de secours pour votre application et est généralement utilisée lorsque la locale préférée d'un utilisateur n'est pas disponible.



<Callout>
    `useDefaultLocale` est un hook côté client et *ne peut être utilisé que sur des composants côté client*.
    Assurez-vous que votre application est enveloppée dans un [`<GTProvider>`](/docs/next/api/components/gtprovider).
</Callout>

Voir [`withGTConfig()`](/docs/next/api/config/withGTConfig) pour la configuration.
Si aucune locale par défaut n'est spécifiée dans [`withGTConfig()`](/docs/next/api/config/withGTConfig), elle est par défaut à `'en-US'`.
Pour le côté serveur, voir [`getDefaultLocale()`](/docs/next/api/helpers/getDefaultLocale).


## Référence

### Retours

Une chaîne représentant la langue par défaut de l'application, par exemple, `'en-US'`.

---

## Exemples

### Utilisation de base

Récupérez la langue par défaut de l'application et affichez-la dans votre composant.

```jsx title="DefaultLocale.jsx" copy
"use client";
import { useDefaultLocale } from 'gt-next/client';

export default function DefaultLocale() {
    const defaultLocale = useDefaultLocale();// [!code highlight]

    return <p>Langue par défaut : {defaultLocale}</p>; // Affichez la langue par défaut
}
```

---

## Notes
 * Le hook `useDefaultLocale()` s'appuie sur le [`<GTProvider>`](/docs/next/api/components/gtprovider) pour accéder au contexte.
   Assurez-vous que votre application est enveloppée avec un fournisseur au niveau racine.
 * `useDefaultLocale()` est uniquement côté client.
 * En savoir plus sur les chaînes de localisation [ici](/docs/reference/locale-strings).

 * Voir [`useLocale`](/docs/next/api/helpers/useLocale) et [`getLocale`](/docs/next/api/helpers/getLocale) pour récupérer la langue de l'utilisateur.
