title: Générer un Modèle Source
description: Comment générer un modèle source pour votre projet
---
## Utilisation

```bash
npx gtx-cli generate
```

## Aperçu

La commande `gtx-cli generate` génère un fichier source pour votre projet dans votre langue par défaut et les langues prises en charge.

Les fichiers générés sont compatibles avec les bibliothèques `gt-next` et `gt-react`, et sont au même format que celui utilisé par la commande `translate`.

Après avoir généré les fichiers sources, vous pouvez utiliser les [traductions locales](/docs/next/reference/local-tx) pour servir les traductions à vos utilisateurs.

<Callout type="info">
    Cette commande n'est utile que si vous utilisez votre propre service de traduction.

    Si vous utilisez l'API General Translation, vous devriez plutôt utiliser la commande `translate`.
</Callout>

## Paramètres

| Paramètre       | Description                                      | Type    | Optionnel | Par défaut         |
|-----------------|--------------------------------------------------|---------|----------|-----------------|
| `--src <paths>`   | Spécifier le(s) répertoire(s) source à scanner                     | `[string]`  | `true`     | `./src && ./app && ./pages && ./components`      |
| `--dictionary <path>`    | Spécifier un chemin vers le fichier dictionnaire                | `string`  | `true`     |                 |
| `--tsconfig, --jsconfig <path>`| Spécifier un chemin vers le fichier de configuration TS ou JS | `string`  | `true`     |                 |
| `--inline`        | Inclure les balises `<T>` en ligne en plus du dictionnaire     | `boolean` | `true`     | `true`            |
| `--default-locale <locale>`| La langue source pour le projet                  | `string`  | `true`     |  `en`               |
| `--ignore-errors` | Ignorer les erreurs et forcer la traduction pour le contenu valide       | `flag` | `true`     | `false`           |
| `-t, --translations-dir <path>`    | Le répertoire où sauvegarder les traductions  | `string`  | `false`     |            |

Le paramètre `-t` est requis, car il spécifie le répertoire où sauvegarder le modèle source.


### Fichier de configuration

Lors de la première exécution de l'outil CLI, il tentera de créer un fichier `gt.config.json` à la racine de votre projet.
Ce fichier contient des métadonnées sur votre projet qui sont utilisées pour traduire votre contenu.

En savoir plus sur le fichier `gt.config.json` [ici](/docs/cli/reference/config).
