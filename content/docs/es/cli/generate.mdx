---
title: Generar Plantilla de Origen
description: Cómo generar una plantilla de origen para tu proyecto
---
## Uso

```bash
npx gtx-cli generate
```

## Descripción general

El comando `gtx-cli generate` genera un archivo de origen para tu proyecto en tu idioma predeterminado y los idiomas compatibles.

Los archivos generados son compatibles con las bibliotecas `gt-next` y `gt-react`, y tienen el mismo formato utilizado por el comando `translate`.

Después de generar los archivos de origen, puedes usar [traducciones locales](/docs/next/reference/local-tx) para proporcionar las traducciones a tus usuarios.

<Callout type="info">
    Este comando solo es útil si estás utilizando tu propio servicio de traducción.

    Si estás utilizando la API de General Translation, deberías usar el comando `translate` en su lugar.
</Callout>

## Parámetros

| Parámetro       | Descripción                                      | Tipo    | Opcional | Predeterminado         |
|-----------------|--------------------------------------------------|---------|----------|-----------------|
| `--src <paths>`   | Especifica el/los directorio(s) de origen para escanear                     | `[string]`  | `true`     | `./src && ./app && ./pages && ./components`      |
| `--dictionary <path>`    | Especifica una ruta al archivo de diccionario                | `string`  | `true`     |                 |
| `--tsconfig, --jsconfig <path>`| Especifica una ruta al archivo de configuración TS o JS | `string`  | `true`     |                 |
| `--inline`        | Incluye etiquetas `<T>` en línea además del diccionario     | `boolean` | `true`     | `true`            |
| `--default-locale <locale>`| El idioma de origen para el proyecto                  | `string`  | `true`     |  `en`               |
| `--ignore-errors` | Ignora errores y fuerza la traducción para contenido válido       | `flag` | `true`     | `false`           |
| `-t, --translations-dir <path>`    | El directorio donde guardar las traducciones  | `string`  | `false`     |            |

El parámetro `-t` es obligatorio, ya que especifica el directorio donde guardar la plantilla de origen.


### Archivo de configuración

Al ejecutar la herramienta CLI por primera vez, intentará crear un archivo `gt.config.json` en la raíz de tu proyecto.
Este archivo contiene metadatos sobre tu proyecto que se utilizan para traducir tu contenido.

Lee más sobre el archivo `gt.config.json` [aquí](/docs/cli/reference/config).
