## Overview

The `getDefaultLocale()` function retrieves the application's default locale.
This locale is determined when the server starts and is used as the fallback language for translations.
It returns a [locale string](/docs/reference/locale-strings), e.g., `'en-US'`.

### When to Use

- Use `getDefaultLocale()` in server-side code where the fallback locale is required.
- Dynamically configure locale-sensitive server-side logic with the application's default locale.

## Reference

### Returns

A string representing the application's default locale, e.g., `'en-US'`.

## Example

### Basic Usage

Retrieve the application's default locale and use it in your server-side logic.

```jsx title="src/server/getDefaultLocale.js" copy
import getDefaultLocale from 'gt-next/server'; // [!code highlight]

export default function logDefaultLocale() {
    const defaultLocale = getDefaultLocale(); // Retrieve the default locale
    console.log(defaultLocale); // Example: 'en-US'

    return defaultLocale;
}
```

## Notes
 * The g`etDefaultLocale()` function is synchronous and provides a reliable default value determined at server startup.
 * The returned locale adheres to the [BCP-47](https://www.techonthenet.com/js/language_tags.php) format. Read more about locale strings [here](/docs/next2/api/reference/locale-strings.mdx).
 * This method is designed for server-side use and complements `getLocale()` for determining the user's active locale.
