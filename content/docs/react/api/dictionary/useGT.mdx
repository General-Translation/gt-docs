## Overview

The `useGT` hook provides access to the `t` translation function from the [`<GTProvider>` context](/docs/react/api/components/gtprovider).
This function enables you to retrieve the translated version of an entry from your [dictionary](/docs/react/reference/dictionaries/dictionaries).

By passing the id of of an entry in your dictionary to the `t` function, you can retrieve the translated value.
The translated value can either be a simple string or an arbitrary JSX structure that can include variables and branching logic.

```jsx
const t = useGT(); // Get the translation function
t('greeting.hello'); // pass the id to get a translation
```

It also supports optional key prefixes for namespace-based template grouping and the insertion of dynamic values into the translation content.


<Callout>
    Using a dictionary design pattern is different from using the `<T>` component.
    To read more about the dictionary pattern, see [this article](/docs/react/reference/dictionaries/dictionaries).

    If you are only using `<T>` components for translation, then this document is not relevant.
</Callout>

## Reference

### Parameters

<TypeTable
  type={{
    "id?": {
        type: 'string',
        optional: true,
        default: 'undefined',
    },
  }}
/>

### Description

| Prop | Description |
| ---- | ----------- |
| `id` | An optional prefix to prepend to all translation keys. This is useful for working with namespaced dictionaries. |

### Returns

A translation function `t` with the following signature:

```jsx
(key: string) => string
```
- `key`: The identifier of the translation to fetch.
- Returns: The translated value as a `string`.

## Examples

### Basic Usages

The `useGT` hook retrieves the `t` function, which can translate any key from your [dictionary](/docs/react/reference/dictionaries/dictionaries).
```jsx title="src/dictionary.jsx" copy
const dictionary = {
    greeting: {
        hello: "Hello, World!" // [!code highlight]
    }
}
export default dictionary;
```
```jsx title="src/components/Greeting.jsx" copy
import { useGT } from 'gt-react';

export default function Greeting() {

    const t = useGT(); // Get the translation function // [!code highlight]

    return (
        <p>
            {t('greeting.hello')} {/* Renders the translated text for 'greeting.hello' */} // [!code highlight]
        </p>
    );
}
```
### Using variables
Variable components ([`<Currency>`](/docs/react/api/components/currency), [`<DateTime>`](/docs/react/api/components/datetime), [`<Num>`](/docs/react/api/components/num), and [`<Var>`](/docs/react/api/components/var))
allow us to inject dynamic values into translations.

We can pass variables to the translation via the `t` function.
We just have to specify an identifier to know where to place the variable in the translation.

```jsx title="src/dictionary.jsx" copy
const dictionary = {
    // 'name-of-user' is used to identify where to place the variable
    greeting: <> Hello, <Var name="name-of-user" /> </> // [!code highlight]
}
export default dictionary;
```

```jsx title="src/components/UserGreeting.jsx" copy
import { useGT } from 'gt-react';

export default function UserGreeting({ user }) {
    const t = useGT();
    return (
        <p>
            {/* use the identifier to pass user.name to the translation */}
            {t('greeting', { 'name-of-user': user.name })}  // [!code highlight]
        </p>
    );
}

```

### Using branches
Branching components, such as [`<Branch>`](/docs/react/api/components/branch) and [`<Plural>`](/docs/react/api/components/plural),
can be used to handle control logic in a translation.

In order to pass values, you must specify a name for the components and reference it when calling the `t()` function.
In this example, we see that `hairColor` is passed to both `<Branch>` and `<Var>` components.
```jsx title="dictionary.jsx" copy
import { Branch, Var } from 'gt-react';

const dictionary = {
  hairColor: <>
    <Branch name="hairColor" // [!code highlight]
      black={<>Their hair is dark.</>}
      brown={<>Their hair is in the middle.</>}
      blonde={<>Their hair is light.</>}
      >
        <p>Unhandled hair color: <Var name="hairColor"/></p> // [!code highlight]
      </Branch>
  </>,
};
```

```jsx title="HairColor.jsx" copy
import { useGT } from 'gt-react';

export default function HairColor(user) {
  const t = useGT();
  return (
    <div>
      { t('hairColor', { hairColor: user.hairColor }) } // [!code highlight]
    </div>
  );
}
```

### Using prefixes

Provide an optional `id` parameter to prepend a prefix to all keys.
```jsx  title="src/dictionary.jsx" copy
import { Var } from 'gt-react';

const dictionary = {
  user: {     // Namespace called user  // [!code highlight]
    name: "John Doe",
    email: "email@example.com" 
  }
}
export default dictionary;
```
```jsx title="src/components/UserDetails.jsx" copy
import { useGT } from 'gt-react';

export default function UserDetails() {
  const t = useGT('user'); // [!code highlight]
  return (
    <div>
      <p>{t('name')}</p> {/* name -> user.name */}
      <p>{t('email')}</p> {/* email -> user.email */}
    </div>
  );
}
```

## Notes
 * The `useGT` hook can only be used within a component wrapped by a [`<GTProvider>` component](/docs/react/api/components/gtprovider). Ensure your application is properly set up with a provider.
