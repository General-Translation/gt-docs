## Overview

The `useElement()` hook allows you to access jsx content from your [translation dictionary](/docs/react/reference/dictionaries/dictionaries).

```jsx
const t = useElement(); // Get the JSX translation function
t('greeting.hello'); // pass the id to get a translation
```

`useElement()` supports:
  * Jsx translation only (no string support).
  * Variable insertion and conditional logic within translations.
  * Optional id prefixing.

<Callout>
  `useGT()` and `useElement()` use a [dictionary](/docs/react/reference/dictionaries/dictionaries) to store all content for translation.
  This is different from using the [`<T>` component](/docs/react/reference/t-reference) for translation.
  If you are interested in only using `<T>` components for translation, then this document is not relevant.
</Callout>


For accessing both strings and jsx content, use [`useGT()`](/docs/react/api/dictionary/useGT).

## Reference

### Props

<TypeTable
  type={{
    "id?": {
        type: 'string',
        optional: true,
        default: 'undefined',
    },
  }}
/>

### Description
| Prop | Description |
| ---- | ----------- |
| `id` | An optional prefix to prepend to all translation keys. This is useful for working with nested dictionary values. |

### Returns

A translation function `t` with the following signature:

```jsx
(key: string) => React.ReactElement
```
- `key`: The identifier of the translation to fetch.
- Returns: The translated value as a `React.ReactElement`.

## Examples

### Basic Usages
Every entry in your dictionary gets translated.
`useElement()` can only resolve jsx entries.
Strings are forbidden.

```jsx title="dictionary.jsx" copy
const dictionary = {
  greeting: <>Hello, Alice</>, // [!code highlight]
};
export default dictionary;
```

When we want to access these entries, we call `useElement()`.
This returns a function that accepts the key of a translation from the dictionary.

```jsx title="translateGreeting.jsx" copy
import { useElement } from 'gt-react';

export default async function translateGreeting() {
  const t = useElement(); // [!code highlight]
  return (
    <p>
      {t('greeting')} // [!code highlight]
    </p>
  );
}
```

### Using variables

You can pass variables to dictionary translations.
Just use a [variable component](/docs/react/reference/dictionaries/variable-components).

<Callout type="error">
  Whenever you have a variable in your jsx translations,
  NEVER insert variables directly into the jsx.
  ALWAYS use a [variable component](/docs/react/reference/dictionaries/variable-components#motive):
  `<Currency>`, `<DateTime>`, `<Num>`, or `<Var>`.
</Callout>

In order to pass values, you must (1) assign an identifier and (2) reference the identifier when calling the `t()` function.
In this example, we use `<Var>` to pass a variable to the translation.

In the dictionary, we assign `<Var>` the identifier `name="userName"`.

```jsx title="dictionary.jsx" copy
const dictionary = {
  greeting: <>Hello, <Var name="userName"/> !</>, // [!code highlight]
};
export default dictionary;
```
In the component, we pass the value by invoking the identifier `userName: "Alice"`.

```jsx title="translateGreeting.jsx" copy
import { useElement } from 'gt-react';

export default async function translateGreeting() {
  const t = useElement();

  return (<p> {t('greeting', { userName: "Alice" })} </p>); // [!code highlight]
}
```

### Using branches
[Branching components](/docs/react/reference/dictionaries/branching-components), such as [`<Branch>`](/docs/react/api/components/branch) and [`<Plural>`](/docs/react/api/components/plural),
can be used to handle control logic in a translation.

In order to pass values, you must (1) assign an identifier and (2) reference the identifier when calling the `t()` function.
In this example, we see that `hairColor` is passed to both `<Branch>` and `<Var>` components.
```jsx title="dictionary.jsx" copy
import { Branch, Var } from 'gt-react';

const dictionary = {
  hairColor: <>
    <Branch name="hairColor" // [!code highlight]
      black={<>Their hair is dark.</>}
      brown={<>Their hair is in the middle.</>}
      blonde={<>Their hair is light.</>}
      >
        {/* fallback incase no branch is matched */}
        <p>Unhandled hair color: <Var name="hairColor"/></p> // [!code highlight]
      </Branch>
  </>,
};
```

```jsx title="HairColor.jsx" copy
import { useElement } from 'gt-react';

export default function HairColor() {
  const t = useElement();
  return (
    <div>
      { t('hairColor', { hairColor: 'brown' }) } // [!code highlight]
    </div>
  );
}
```

### Using prefixes

We can use prefixes to only translate a subset of the dictionary.
```jsx  title="dictionary.jsx" copy
const dictionary = {
  prefix1: { // [!code highlight]
    prefix2: { // [!code highlight]
      greeting: <>Hello, Alice</>,
    }
  }
};
export default dictionary;
```
Because we added the value `'prefix1.prefix2'` to the `useElement()` hook, all of the keys are prefixed with `prefix1.prefix2`:
```jsx title="UserDetails.jsx" copy
import { useElement } from 'gt-react';

export default function UserDetails() {
  const t = useElement('prefix1.prefix2'); // [!code highlight]
  return (
    <div>
      <p>{t('greeting')}</p> {/* greeting1 => prefix1.prefix2.greeting1 */} // [!code highlight]
    </div>
  );
}
```

## Notes
 * `useElement()` only translates JSX content from the dictionary. No strings.
 * For string-based translations, consider using [`useGT()`](/docs/react/api/dictionary/useGT).
