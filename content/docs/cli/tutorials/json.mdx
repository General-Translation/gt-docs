## Overview

If you are using a 3rd party i18n library, [stand alone `gt-next`](/docs/next/reference/stand-alone), stand alone `gt-react`, or a custom i18n implementation, you can automatically update all of your translation JSON files every time your project changes.

Just add the [`gtx-cli translate`](/docs/cli/translate#translate-json) command to your build or CI process before the build command to automatically add translations to your project.

```json title="package.json"
{
  "scripts": {
    "translate": "npx gtx-cli translate",
    "build": "npm run translate && <your build command>"
  }
}
```

Then add your production API key and project ID to your environment variables.
This is necessary to use the `gtx-cli` tool.
You get these from the [General Translation dashboard](https://generaltranslation.com/dashboard).

```bash title=".env"
GT_API_KEY=<your-api-key>
GT_PROJECT_ID=<your-project-id>
```


<Callout type="info">
  **Note:**
  We currently support custom string syntax and formatting for the following i18n libraries: [`next-intl`](/docs/cli/tutorials/next-intl-quickstart), [`react-i18next`](/docs/cli/tutorials/react-i18next-quickstart), [`i18next`](/docs/cli/tutorials/i18next-quickstart), `gt-next`, `gt-react`.
  
  If you are using a different i18n library, the translate command will only work for plain strings.
  Don't see your favorite library? [Please let us know](https://github.com/General-Translation/gt-libraries/issues), and we will add it as soon as we can!
</Callout>

## Notes
 * You can automatically add translations to your project by with the [`gtx-cli translate`](/docs/cli/translate#translate-json) command.
 * If you want to commit your translation files, you can instead run the `gtx-cli translate` command before committing.
